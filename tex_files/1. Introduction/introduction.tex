\section{Introduction}
\label{sec:intro}
 
A literature search asks two things of a retrieval system: firstly, to find the papers that precisely match a set of stated constraints; secondly, to find the papers that are about what the user means. These map onto two retrieval paradigms that have proven difficult to reconcile. Structured retrieval, being exact keyword and Boolean matching over typed metadata, answers a query deterministically: a paper either satisfies the constraints or it does not, and the same query returns the same answer. Semantic retrieval, being dense vector embeddings that rank documents by meaning, recovers relevance that exact terms miss, but returns what lies nearest in a learned space rather than what provably matches. The two serve different needs. Assembling an evidence base, as in a systematic review, demands the determinism and auditability of the former; exploring an unfamiliar area rewards the recall of the latter. Deployed literature-search engines force a choice between them.
 
This thesis addresses that gap in a deployed setting. The work was conducted with an industry partner, Latent Knowledge, whose users are biomedical researchers, and the system it documents is in production: a typed scholarly knowledge graph of approximately 85 million biomedical works, built from OpenAlex and queried through a three-stage retrieval cascade, has served a limited user base since the partner's v1 launch. % VERIFY: confirm the production v1 configuration runs all three retrieval stages, and whether a launch month should be named.
The deployment shapes the research in two ways. It grounds every model choice in the constraints of a real system, where memory footprint, query latency, and licensing weigh against raw accuracy, and it fixes the corpus at biomedical literature while the schema and architecture remain general purpose.
 
The aim of this thesis is therefore to design, build, and evaluate a scholarly search system that reconciles the two paradigms rather than choosing between them. Retrieval proceeds in three stages: structured graph retrieval over the knowledge graph produces a candidate set that provably satisfies the query's nested Boolean constraints; bi-encoder dense retrieval ranks that set by semantic similarity; and cross-encoder re-ranking orders the final results. Three research questions structure the evaluation, one for each layer of the design:
 
\begin{enumerate}
    \item To what extent can modern language models construct the two artefacts that deterministic structured graph retrieval requires, namely the per-work keyword index extracted offline from the corpus, and the intermediate representation translated from each natural-language query?
    \item How well do current bi-encoders and cross-encoders perform semantic ad-hoc search on scientific retrieval benchmarks?
    \item How do the results of the three-stage cascade differ from those of existing search systems, and what explains the differences?
\end{enumerate}
 
In answering these, the thesis makes four contributions: (i) a typed scholarly knowledge graph of approximately 85 million works, with a documented schema and ingestion pipeline from the OpenAlex snapshot; (ii) a natural-language query interface that decouples LLM-based intent extraction from deterministic Cypher compilation through a JSON intermediate representation; (iii) a hand-built, stratified benchmark for the natural-language-to-IR translation task, which no existing benchmark covers; and (iv) component-level evaluations of keyword extractors, bi-encoders, and cross-encoders that select the deployed models and make the accuracy and deployment trade-offs of each choice explicit.
 
The remainder of this report is organised as follows. Section~\ref{sec:litreview} reviews the literature on scholarly retrieval, positioning the design against dense retrieval, graph-augmented methods, and structured-query translation, and surveying the systems and benchmarks currently available. Section~\ref{sec:research-design} presents the research design: the system architecture, the graph schema and ingestion pipeline, the design of each retrieval stage, and the methodology used to validate them. Section~\ref{sec:data-results} reports the outcome of each component benchmark, which Section~\ref{sec:analysis-discussion} then analyses against the three research questions. Section~\ref{sec:conclusion} draws conclusions and identifies future work.
